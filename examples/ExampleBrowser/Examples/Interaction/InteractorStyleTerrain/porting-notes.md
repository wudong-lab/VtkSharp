## Added API

- vtkInteractorStyleTerrain (new class, vtkInteractionStyle) — full supported API
- vtkCamera::Azimuth(double)
- vtkCamera::Elevation(double)
- vtkRenderWindowInteractor::SetInteractorStyle(vtkInteractorObserver*)

## Unsupported / Skipped

The following methods were present in the candidate generated by `create-candidate`
but manually removed from the final whitelist due to generation issues:

- vtkCommand*-based methods (GrabFocus, HasObserver, RemoveObserver, RemoveObservers) —
  vtkCommand is a static class (hand-written) and cannot be used as a parameter type
- operator new / operator delete — not valid for C# bindings
- OnDropLocation(double*) — primitive pointer parameter without direction metadata
- Duplicate NewInstance overrides (vtkInteractorObserver*, vtkInteractorStyle*, vtkObject*) —
  only the vtkInteractorStyleTerrain* variant is needed; others cause C2556 compiler errors

## Deviations from Original

- `colors->GetColor3d("MistyRose").GetData()` → `colors.GetColor3d("MistyRose")` then
  pass `.R`, `.G`, `.B` to `actor.GetProperty().SetColor()`
- `colors->GetColor3d("SlateGray").GetData()` → same pattern for `renderer.SetBackground()`
